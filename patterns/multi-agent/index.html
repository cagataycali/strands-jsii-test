    Multi-Agent Patterns¶

    Orchestrate multiple agents for complex workflows.

    Agent-as-Tool (Recommended)¶

    The simplest multi-agent pattern — wrap agents as tools:

    from strands_agents_jsii import StrandsAgent, AgentConfig, AgentTool, BedrockModelProvider, BedrockModelConfig

def make_agent(prompt):
    return StrandsAgent(AgentConfig(
        model=BedrockModelProvider(BedrockModelConfig("us.anthropic.claude-sonnet-4-20250514-v1:0")),
        system_prompt=prompt,
    ))

# Specialists
sql_agent = make_agent("You write SQL queries. Return only the query.")
python_agent = make_agent("You write Python code. Return only the code.")
docs_agent = make_agent("You write documentation. Be thorough.")

# Manager
manager = StrandsAgent(AgentConfig(
    model=BedrockModelProvider(BedrockModelConfig("us.anthropic.claude-sonnet-4-20250514-v1:0")),
    system_prompt="Route tasks to the right specialist.",
    tools=[
        AgentTool("sql_expert", "Write SQL queries", sql_agent),
        AgentTool("python_expert", "Write Python code", python_agent),
        AgentTool("docs_writer", "Write documentation", docs_agent),
    ],
))

response = manager.invoke("Create a user table in SQL, Python ORM model, and API docs")

    Pattern Comparison¶

    1. Router (Manager delegates)¶

    graph TD
    U[User] --> M[Manager]
    M -->|task A| A[Agent A]
    M -->|task B| B[Agent B]
    A --> M
    B --> M
    M --> U

    Best for: Tasks with clear domain boundaries

    2. Pipeline (Sequential handoff)¶

    graph LR
    U[User] --> A[Research] --> B[Draft] --> C[Edit] --> U
    # Each step feeds into the next
research = manager.invoke("Research topic X")
draft = manager.invoke(f"Write article from: {research.text}")
final = manager.invoke(f"Edit this draft: {draft.text}")

    Best for: Content creation, data processing pipelines

    3. Debate (Multiple perspectives)¶

    # Get opposing views, then synthesize
response = manager.invoke(
    "Get the optimist, pessimist, and realist views on AI regulation, "
    "then write a balanced summary."
)

    Best for: Decision making, analysis

    Nesting Depth¶

    Agents can be nested multiple levels:

    CEO Agent
├── CTO Agent (tool)
│   ├── Backend Agent (tool)
│   └── Frontend Agent (tool)
└── CFO Agent (tool)
    └── Analytics Agent (tool)

    Set max_cycles

    Always set max_cycles on agents to prevent infinite delegation loops. A good default is 5-10 for inner agents, 20-50 for outer agents.
