    The Agent Loop¶

    The core agent loop runs inside the TypeScript SDK, managed by jsii:

    sequenceDiagram
    participant App as Your Code (Python/Java/Go/C#)
    participant jsii as jsii Runtime
    participant Agent as Agent Loop (TypeScript)
    participant Model as Model Provider
    participant Tool as Your Tool (Python/Java/Go/C#)

    App->>jsii: agent.invoke("query")
    jsii->>Agent: process(messages)

    loop Until no more tool_use
        Agent->>Model: converse(messages)
        Model-->>Agent: response (text + tool_use)

        opt If tool_use
            Agent->>jsii: tool.execute(input)
            jsii->>Tool: execute(input_json)
            Tool-->>jsii: result_json
            jsii-->>Agent: tool_result
        end
    end

    Agent-->>jsii: final response
    jsii-->>App: AgentResponse

    Key Design Decisions¶

    1. Synchronous Model Calls via Subprocess¶

    Model providers use child_process.execSync to call AWS CLI or curl:

    Agent Loop → execSync("aws bedrock-runtime converse ...") → JSON response

    This is intentionally synchronous because jsii doesn't support async/await across language boundaries. The trade-off: simpler API, no callback hell.

    2. Tools Execute in Your Language¶

    When the model requests a tool call, the jsii runtime calls back into your language:

    TypeScript Agent → jsii callback → Python ToolDefinition.execute() → jsii → TypeScript

    Your tool code runs natively in Python/Java/Go/C#. No serialization overhead beyond JSON.

    3. Single TypeScript Source → 4 Language Bindings¶

    src/agent.ts + src/model.ts
    ↓ jsii compile
.jsii assembly (type metadata)
    ↓ jsii-pacmak
dist/python/  dist/java/  dist/go/  dist/dotnet/

    Comparison with Python SDK¶

    Aspect Python SDK jsii SDK
    Languages Python only Python, Java, Go, C#
    Model calls Async (boto3, httpx) Sync (subprocess)
    Streaming ✅ Full streaming ❌ Full response only
    Tool execution Python native Native in each language
    Agent loop Python TypeScript (via jsii)
    Dependencies pip packages per provider Node.js + jsii only
    Performance Optimal ~50ms overhead per model call
