# strands-jsii Deep Dive - 2026-03-03

## Architecture Overview
- **jsii layer** (src/): TypeScript compiled via jsii for Python/Java/C#/Go bindings
- **web layer** (src/web/): Browser-compatible async agents with streaming, excludeTypescript from jsii
- **Shared formats** (src/providers/formats.ts): Single source of truth for ALL provider request/response formatting

## Key Architectural Decisions
1. **Sync jsii constraint**: All model providers use execSync+curl or subprocess spawning
2. **Bedrock uses AWS SDK** via spawned Node.js scripts (with streaming support)
3. **Other providers use curl** for HTTP calls (Anthropic, OpenAI, Gemini, Ollama)
4. **Web layer uses fetch()** - pure async, zero Node.js deps
5. **formats.ts is shared** - build/parse functions used by BOTH jsii models and web providers

## Missing: OpenTelemetry
- NO telemetry/tracing code exists in strands-jsii yet
- Python SDK has comprehensive OTEL: tracer.py, config.py, metrics.py
- Need spans for: agent invocation, model calls, tool execution, event loop cycles
- Need metrics: token usage, latency, tool call counts, cycle durations
- Langfuse compatibility needed (is_langfuse check in Python SDK)

3. Ollama: tool_calls formatting uses toolUseId as function name instead of actual name
4. SlidingWindow: Always keeps first message but slices from end - could break tool result pairs
5. RetryStrategy: Uses execSync('sleep') which is hacky
